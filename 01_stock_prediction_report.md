# 주식 등락률 예측 AI 모델 개발 — 결과 보고서

> **KOSPI / KOSDAQ 시총 구간별 층화 샘플링 기반 9개 회귀 모델 비교 분석**
> LSTM · Random Forest · XGBoost · 선형 회귀 · Ridge · Lasso · ElasticNet · 결정 트리 · KNN
> 포트폴리오 프로젝트 · 2025

---

## 1. 프로젝트 개요

### 1.1 연구 배경 및 목적

국내 주식시장(KOSPI/KOSDAQ)의 2,700여 종목 전체를 대상으로 기계학습 모델의 단기 등락률 예측 가능성을 검증하고,
**9가지 회귀 모델**(LSTM · Random Forest · XGBoost · 선형 회귀 · Ridge · Lasso · ElasticNet · 결정 트리 · KNN)을 동일한 조건에서 비교 분석하였다.

단일 종목 분석의 한계를 극복하기 위해 **다중 종목 일반화 모델**을 구현하였으며,
2025년 실전 데이터를 이용해 Out-of-Sample 성능을 측정함으로써 모델의 실용성을 평가하였다.

### 1.2 데이터 범위

| 항목 | 내용 |
|------|------|
| 수집 종목 수 | 2,731개 (KOSPI + KOSDAQ 전 종목) |
| **학습 종목 수** | **200개 (시총 구간별 층화 샘플링)** |
| 학습 기간 | 2019-01-02 ~ 2024-12-31 |
| 검증 기간 | 2025-01-02 ~ 현재 |
| 입력 특성 | 11개 기술적 지표 |
| 입력 시퀀스 | 과거 60거래일 |
| 예측 대상 | 5거래일 후 등락률 |

### 1.3 학습 종목 선정 전략 — 시총 구간별 층화 샘플링

단순히 시총 상위 20개 대형주만 학습하면 **소형주 패턴을 전혀 반영하지 못하는 편향된 모델**이 된다.
반대로 전 종목(2,731개)을 학습하면 약 8 GB의 메모리가 필요해 일반 환경에서 실행이 불가능하다.

이 두 한계를 동시에 해결하기 위해 **층화 샘플링(Stratified Sampling)** 을 채택하였다.

> **층화 샘플링이란?** 모집단을 의미 있는 기준으로 여러 그룹(층)으로 나눈 뒤,
> 각 층에서 동일한 수를 무작위 추출해 전체 분포를 균등하게 대표하는 표본을 구성하는 방법.
> 설문조사·임상시험 등에서 표준으로 사용하는 통계적 샘플링 기법이다.

#### 구간 설계

| 구간 | 시총 범위 | 샘플 수 | 대표 종목 예시 |
|------|-----------|---------|--------------|
| 대형주 (상위 25%) | 상위 683개 중 50개 | 50개 | 삼성전자, SK하이닉스, LG화학 |
| 중대형주 (50~75%) | 중상위 682개 중 50개 | 50개 | 현대차, KB금융, 포스코홀딩스 |
| 중소형주 (25~50%) | 중하위 683개 중 50개 | 50개 | 대원제약, 율촌화학 |
| 소형주 (하위 25%) | 하위 683개 중 50개 | 50개 | KR모터스, 영흥, 블루산업개발 |
| **합계** | | **200개** | |

#### 이 방식을 선택한 이유

1. **통계적 대표성**: 전체 시장의 규모별 분포를 고르게 반영 → 특정 규모 종목에 편향되지 않음
2. **다양한 패턴 학습**: 대형주(지수 연동·외국인 영향)와 소형주(수급·테마 영향)는 주가 움직임 패턴이 근본적으로 다름
3. **메모리 효율**: 200종목 × 약 1,447 시퀀스(6년치 거래일 기준) = 약 29만 샘플 (~0.8 GB) → 일반 PC에서 실행 가능
4. **재현 가능성**: `random_state=42` 고정으로 항상 동일한 200종목 선택

## 2. 기술 스택 및 아키텍처

### 2.1 데이터 수집 파이프라인

- **FinanceDataReader**: KOSPI/KOSDAQ 전 종목 일별 OHLCV 수집
- **ThreadPoolExecutor**: 병렬 다운로드로 수집 시간 단축
- **Parquet 포맷 캐싱**: 재실행 시 디스크 캐시 활용

### 2.2 입력 설계 근거

#### ① 시퀀스 길이: 왜 60거래일인가?

60거래일은 약 **3개월치 거래일**에 해당한다. 이 값을 선택한 근거는 다음과 같다.

| 후보 길이 | 의미 | 판단 |
|-----------|------|------|
| 20일 (1개월) | 단기 노이즈 위주, 추세 파악 부족 | 부족 |
| **60일 (3개월)** | **단기·중기 패턴 동시 포착 가능** | **✅ 선택** |
| 120일 (6개월) | 더 많은 맥락 확보, 그러나 샘플 수 감소 & 오래된 정보 비중 증가 | 과도 |

- 기술적 분석에서 **60일 이동평균(MA_60)**은 중기 추세의 기준선으로 널리 사용된다.  
  60일 시퀀스를 입력으로 쓰면 모델이 MA_60이 형성되는 전 과정을 직접 학습할 수 있다.
- RSI·MACD 같은 모멘텀 지표는 **14~26일 윈도우**로 계산되므로,  
  60일 시퀀스면 해당 지표가 의미 있는 수준으로 수렴하는 구간을 충분히 포함한다.
- 샘플 수와 맥락 길이의 **트레이드오프**: 시퀀스가 길수록 샘플 수가 줄고 학습 시간이 늘어난다.  
  60일은 200개 학습 종목에서 충분한 샘플(약 29만 개, 종목당 ~1,447 시퀀스)을 확보하면서 맥락도 유지하는 균형점이다.

#### ② 특성 수: 왜 11개인가?

단순히 많은 지표를 넣는 것이 성능을 보장하지 않는다.  
**정보 중복 최소화 + 서로 다른 시장 국면 포착**을 기준으로 11개를 선별하였다.

| 분류 | 포함 지표 | 포착하는 정보 |
|------|-----------|--------------|
| 가격 | Close | 절대 가격 수준 (정규화 후) |
| 거래량 | Volume, Volume_Ratio | 매매 참여도, 이상 거래 감지 |
| 모멘텀 | RSI, MACD, MACD_Signal | 추세 강도·방향·전환 신호 |
| 변동성 | BB_Pct, BB_Width, ATR | 가격 밴드 위치, 시장 변동성 수준 |
| 추세 | MA_20, MA_60 | 단기·중기 추세 방향 |

- **RSI와 MACD는 중복이 아닌가?**: RSI는 *절대적 과매수/과매도*, MACD는 *추세 전환 타이밍*을 측정해 상호 보완적이다.
- **OHLCV 중 종가(Close)만 사용한 이유**: 시가·고가·저가는 종가와 상관관계가 매우 높아 정보 중복이 크다. 대신 그 변동성은 ATR(Average True Range)이 요약해 포함한다.
- **재무제표·뉴스 미포함**: 본 프로젝트는 일별 기술적 지표만으로 단기 패턴을 학습하는 것을 목표로 하였으며, 비정형 정보 통합은 개선 방향으로 제시하였다(7장 참조).

### 2.3 기술적 지표 (11개)

| 지표 | 설명 |
|------|------|
| Close | 종가 (정규화) |
| Volume | 거래량 |
| RSI (14일) | 과매수/과매도 판단 |
| MACD | 추세 전환 신호 |
| MACD_Signal | MACD 9일 EMA |
| BB_Pct | 볼린저밴드 내 위치 |
| BB_Width | 볼린저밴드 폭 (변동성) |
| MA_20 | 20일 이동평균 |

| ATR (14일) | 평균 진폭 (변동성) |
| Volume_Ratio | 거래량 / 20일 평균거래량 |

### 2.4 모델 개념 설명

본 프로젝트에서 사용한 9개 모델은 접근 방식이 근본적으로 다르다.
각 모델이 **어떤 원리로 예측하는지** 이해하면 결과 해석이 쉬워진다.

---

#### ① LSTM (Long Short-Term Memory) — 흐름을 기억하는 딥러닝

**핵심 아이디어**: 사람이 차트를 볼 때처럼, 과거 60일의 흐름을 순서대로 기억하면서 패턴을 학습한다.

일반 신경망은 입력을 독립적으로 처리해 시간 순서를 무시하지만,
LSTM은 **이전 시점의 정보를 다음 시점으로 전달하는 '메모리 셀'** 을 갖고 있다.

| 게이트 | 역할 | 주가 예시 |
|--------|------|-----------|
| 망각 게이트 (Forget) | 과거 기억 중 잊을 것 결정 | "3달 전 급등은 지금과 무관 → 잊기" |
| 입력 게이트 (Input) | 현재 입력 중 기억할 것 결정 | "오늘 RSI 30 돌파 → 중요, 기억" |
| 출력 게이트 (Output) | 지금 기억에서 출력할 것 결정 | "기억을 바탕으로 5일 후 등락률 출력" |

- **장점**: 60일치 흐름 전체를 보고 판단 → 시간 패턴 학습 가능
- **단점**: 학습 시간이 길고, "왜 이렇게 예측했는지" 설명이 어려움 (블랙박스)

---

#### ② Random Forest — 여러 전문가의 다수결

**핵심 아이디어**: 300명의 분석가가 각자 다른 데이터를 보고 예측한 뒤, 평균을 낸다.

```
학습 데이터 → 랜덤 샘플링 300번 → 결정 트리 300개 독립 학습
예측 시: 300개 트리가 각자 등락률 예측 → 평균 → 최종 예측
```

- **배깅(Bagging)**: 각 트리가 서로 다른 데이터로 학습 → 다양한 관점 형성
- **장점**: 학습 빠름(~30초), 특성 중요도 제공으로 해석 가능, 과적합 강건
- **단점**: 오늘 하루 데이터만 입력 → 시간 흐름 패턴 학습 불가

---

#### ③ XGBoost (eXtreme Gradient Boosting) — 오답 노트 학습

**핵심 아이디어**: 이전 모델이 틀린 부분을 다음 모델이 집중적으로 보완한다.

```
1번 트리: 전체 예측 → 오차 계산
2번 트리: 1번 트리의 오차를 타깃으로 학습
3번 트리: 1+2번 합산 오차를 타깃으로 학습
...500번까지 반복 → 최종 예측 = 전체 트리 합산
```

- **RF와의 차이**: RF는 트리를 병렬로 독립 생성, XGB는 순차적으로 오차 보정
- **Early Stopping**: 30라운드 개선 없으면 자동 종료 → 과적합 방지
- **장점**: 높은 정확도, 빠른 학습, 특성 중요도 제공
- **단점**: RF보다 하이퍼파라미터 민감, 시간 순서 무시

---

---

#### ④ 선형 회귀 (Linear Regression) — 가장 단순한 기준선

**핵심 아이디어**: 입력 지표 × 가중치의 합으로 등락률을 예측한다.

- **장점**: 가장 단순하고 빠름. 다른 모델의 성능 기준선(Baseline) 역할
- **단점**: 지표 간 복잡한 비선형 관계를 포착하지 못함

---

#### ⑤ Ridge / Lasso / ElasticNet — 규제를 추가한 선형 회귀

**핵심 아이디어**: 선형 회귀에 가중치를 억제하는 패널티를 추가해 과적합을 방지한다.

| 모델 | 규제 방식 | 특징 |
|------|-----------|------|
| **Ridge** | 계수 제곱합 최소화 | 모든 계수를 조금씩 줄임 (완전히 0이 되지 않음) |
| **Lasso** | 계수 절댓값 합 최소화 | 중요하지 않은 계수를 정확히 0으로 만듦 → 자동 변수 선택 |
| **ElasticNet** | Ridge + Lasso 혼합 | 두 방식의 장점을 균형 있게 결합 |

- **Alpha(α)**: 규제 강도. 클수록 계수가 더 강하게 억제됨. 탐색으로 최적값 결정.

---

#### ⑥ 결정 트리 (Decision Tree) — 조건 분기로 예측

**핵심 아이디어**: "RSI < 30 이면서 거래량이 평균보다 높으면 → 상승 예측"처럼 조건 분기를 반복한다.

- **장점**: 예측 규칙을 사람이 직접 읽을 수 있어 해석성이 매우 높음
- **단점**: 깊이가 깊어질수록 훈련 데이터만 외움 (과적합) → `max_depth`로 제한

---

#### ⑦ KNN 회귀 (K-Nearest Neighbors) — 비슷한 과거 데이터의 평균

**핵심 아이디어**: "오늘과 가장 비슷했던 과거 k개 날을 찾아, 그 날들 이후의 등락률 평균을 예측"

- **장점**: 별도 학습 과정 없이 직관적. 데이터가 많을수록 성능 향상.
- **단점**: 예측 시마다 전체 데이터를 검색 → 예측 속도 느림. 고차원 데이터에 취약.

---

### 2.5 LSTM 모델 아키텍처

```
입력: (N, 60, 11)
  └─ LSTM 128유닛 (return_sequences=True)
     └─ BatchNormalization + Dropout(0.2)
        └─ LSTM 64유닛 (return_sequences=True)
           └─ BatchNormalization + Dropout(0.2)
              └─ LSTM 32유닛
                 └─ Dropout(0.1)
                    └─ Dense 16 (ReLU)
                       └─ Dense 1 (출력: 등락률)
```

- **손실함수**: Huber Loss (이상치에 강건)
- **옵티마이저**: Adam (lr=0.001)
- **총 파라미터**: 134,817개
- **Early Stopping**: patience=7

## 7. 한계 및 개선 방향

### 7.1 현재 한계

| 구분 | 한계 |
|------|------|
| 과적합 위험 | 시간 순서 미반영 Train/Test 분리, 스케일러 데이터 누수 |
| 정보 부재 | 기술적 지표만 사용 (뉴스·공시·거시지표 미반영) |
| 전략 단순성 | 거래 비용 미반영, 포트폴리오 최적화 미적용 |

### 7.2 개선 방향

1. **Walk-Forward 검증**: 시간 순서를 유지한 Rolling 방식 적용
2. **멀티모달 입력**: KoBERT 뉴스 감성 + 거시경제 변수 결합
3. **강화학습 확장**: 동적 포트폴리오 배분 (DQN / PPO)
4. **거래 비용 통합**: 수수료·세금·슬리피지 모델 반영
5. **앙상블**: LSTM + XGBoost 가중 앙상블로 개별 모델 약점 보완

## 8. 결론

본 프로젝트는 KOSPI/KOSDAQ 전 종목 수집 후 **시총 구간별 층화 샘플링으로 200개 종목을 선정**하여
**9가지 회귀 모델**(LSTM · RF · XGBoost · 선형 회귀 · Ridge · Lasso · ElasticNet · 결정 트리 · KNN)의 단기 등락률 예측 성능을 체계적으로 비교하였다.

**핵심 결론**:
- 9개 모델 모두 방향 정확도 49~54% 수준으로, 단기 주가 예측의 근본적 한계를 정량적으로 확인하였다
- 시계열 정보를 활용한 LSTM이 MAE·방향정확도 양면에서 가장 우수하였다
- 선형 모델(Ridge·Lasso·ElasticNet)은 학습 시간 <1초임에도 앙상블에 근접한 성능을 보여 효율성이 높았다
- 결정 트리는 과적합 갭(+0.7%p)이 가장 커 일반화 성능에서 상대적으로 취약하였다

2025년 실전 검증에서 대부분 모델의 MAE 증가폭이 0.3~0.5%p에 그쳐 **심각한 과적합 없이 일반화가 이루어졌음**을 확인하였다.
향후 비정형 정보(뉴스·공시) 통합과 Walk-Forward 검증 체계 구축을 통해 실용성을 높일 수 있을 것이다.
